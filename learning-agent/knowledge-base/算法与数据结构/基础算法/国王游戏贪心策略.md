## 核心要点

国王游戏是一个经典的「邻项交换法」贪心问题。核心结论是 **按 A×B 从小到大排序** 可以使最大金币数最小。证明依赖比较相邻两人交换前后的最大值，利用 max 函数的性质建立偏序关系。实现时必须使用**高精度**，因为乘积会快速增长超出 long long 范围。

---

## 一、题目背景

P1080 国王游戏（洛谷）：
- 国王和 n 个大臣各有一个左手数 A 和一个右手数 B
- 国王始终站在队伍第一位
- 第 k 个大臣获得的金币为：

```
\text金币ₖ=A₀×A₁×⋯×A_(k-1)
───────────────
      Bₖ       
```
<!-- $\text金币ₖ=A₀×A₁×⋯×A_(k-1) / Bₖ$ -->

- **目标**：排列大臣顺序，使所有大臣中获得金币的**最大值尽可能小**

---

## 二、贪心策略

### 排序准则

**按 Aᵢ×Bᵢ 从小到大排列大臣。**

| 条件 | 结果 |
|------|------|
| Aᵢ×Bᵢ<Aⱼ×Bⱼ | i 排在 j 前面 |
| Aᵢ×Bᵢ=Aⱼ×Bⱼ | 任意顺序，不影响最终答案 |
| Aᵢ×Bᵢ>Aⱼ×Bⱼ | j 排在 i 前面 |

---

## 三、交换论证证明（核心推导）

- S = i 之前所有人的左手乘积

- Aⱼ,Bⱼ：j 的左右手数

### 3.3 要证明的命题

> 若 Aᵢ·Bᵢ>Aⱼ·Bⱼ，则 M₁>M₂（即交换后更优）

由前提 Aᵢ·Bᵢ>Aⱼ·Bⱼ 可得：


### 3.4 分情况讨论

**情况一**：\displaystyleAᵢ / Bⱼ\ge1 / Bᵢ

则 M'₁=Aᵢ / Bⱼ，而

```
M'₁=Aᵢ
──
Bⱼ\gemax [](1

──
Bᵢ)=M'₂
```
<!--

故 M₁\geM₂。

**情况二**：\displaystyleAᵢ / Bⱼ<1 / Bᵢ

由 Aᵢ / Bⱼ<1 / Bᵢ 推出 Aᵢ·Bᵢ<Bⱼ，结合前提 Aᵢ·Bᵢ>Aⱼ·Bⱼ 可得 Bᵢ<Bⱼ。

进而：
- 1 / Bᵢ>1 / Bⱼ
- Aⱼ / Bᵢ<Aᵢ / Bⱼ<1 / Bᵢ

因此 M'₁=1 / Bᵢ>M'₂，即 M₁>M₂。

### 3.5 结论

**两种情况均表明**：Aᵢ·Bᵢ 大的人应该往后放。对队伍施行任意次相邻交换后，最终顺序必为按 A×B 升序排列，即为全局最优解。

> 这里依赖了 **max 函数的单调性**——max 能保证偏序关系的传递性（若 a > b 且 b > c 则 a > c），这是贪心策略成立的关键。后文会看到，换成 min 就不行了。

## 四、算法流程

```
1. 读入 n, A₀, B₀（国王）
2. 读入 n 个大臣的 (Aᵢ, Bᵢ)
3. 按 Aᵢ × Bᵢ 升序排列
4. product = A₀          ← 高精度
5. ans = 0               ← 高精度
6. for i = 1 to n:
     cur = product / Bᵢ   ← 高精度除法
     ans = max(ans, cur)
     product = product × Aᵢ  ← 高精度乘法
7. 输出 ans
```

---

## 五、关键坑点

### 坑点 1：乘积爆炸

- Aᵢ\le10000，n\le1000，最坏情况下乘积约为 10000^(1000)，远超 long long（约 2^(63)）甚至 `__int128`
- **必须使用高精度**：一般用数组模拟，每位存一个十进制位，或压位存储以提高效率

### 坑点 2：最大值不一定在最后一个人

- 金币 =\text累积乘积 / \text当前B，而 B 的波动可能使中间某人的金币更大
- 因此必须每步都 `ans = max(ans, cur)`，不能只算最后一个人

### 坑点 3：相等情况

- A×B 相等的两人交换不影响最终答案，随意排列即可

## 六、延伸思考：如果改成「最大化最小值」？

### 为什么 min 下交换论证失效

当目标函数从 max 变成 min 时，交换论证**不再成立**。问题出在：

- 对于 max：若 a > b 且 b > c，则 a > c ✓（传递性）
- 对于 min：**不存在这种直接的偏序关系**

min 不能像 max 那样建立稳定的偏序关系——交换两个人的顺序，min 的行为在不同区间可能完全相反，无法通过相邻比较推导出全局序。

### 反例验证

设两人：A₁=2,B₁=3 和 A₂=2,B₂=3（即 A×B 相等），但 B 不同时，按 B 小的站前面更好。这说明当目标变为 min 时，排序准则与 max 场景不同，且无法用简单的单一指标排序。

### 解决方法

最大化最小值的变体问题无法用简单贪心解决，通常需要：
- **二分答案** + 验证可行排列
- **动态规划**（状态压缩）

---

## 一句话总结

A×B 小的站前面，交换论证证明贪心正确——但仅限 max，min 不可贪。

## 关键概念清单

- **邻项交换法**：通过分析相邻两人交换前后的目标函数，建立排序偏序关系的证明技术
- **max 的传递性**：max 函数能保证偏序的传递（a>b\landb>c⇒a>c），是贪心正确性的数学基础
- **高精度**：用数组模拟大整数运算，解决乘积超出标准类型范围的问题
- **贪心的正确性证明**：不能凭直觉，必须通过交换论证或拟阵等严格方法证明
