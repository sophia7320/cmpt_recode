# 错误分析框架

**理论基础**：基于认知诊断理论 (Leighton & Gierl, 2007) 与形成性评估研究 (Black & Wiliam, 1998)。

## 核心理念

错误不是坏事。每个错误都是一条线索，暴露了理解链中的断裂点。关键是：不要只看"错了什么"，而要理解"为什么错"。

## 四步分析流程

### 步骤 1：定位错因
用户答错后，不要急着纠正，先分析：
- 用户说了什么？与正确答案的差异在哪？
- 是答案全错还是部分错误？
- 是犹豫不确定还是坚定确信？

### 步骤 2：错误分类
将错误归入四种类型之一：

| 错误类型 | 描述 | 识别信号 | 示例 |
|----------|------|----------|------|
| **概念型错误** | 对基本概念理解有误 | 定义错误、混淆概念、因果倒置 | "导数就是切线"（缺失"斜率"） |
| **程序型错误** | 知道概念但步骤出错 | 跳过关键步骤、顺序错误 | 求导时忘记链式法则 |
| **迁移型错误** | 无法将知识应用到新场景 | 换个问法就不会、只认例题形式 | 会求 f(x) 极值，不会求 g(x,y) 极值 |
| **遗漏型错误** | 相关知识完全未掌握或遗忘 | 完全空白，无法作答 | 没见过/学了但忘了某个概念 |

### 步骤 3：归纳模式
看回用户最近几次评估/学习中的错误：
- 是否多次出现同类型错误？
- 是否存在系统性误解？（如总是把"XX和YY"搞反）
- 是基础层断裂还是应用层问题？

### 步骤 4：针对性策略
根据错误类型给出纠正方向：

| 错误类型 | 纠正策略 |
|----------|----------|
| 概念型 | 回到基础概念，用费曼技巧重新讲解，消除混淆 |
| 程序型 | 拆解步骤 + 示范 → 让用户练一道同类题 |
| 迁移型 | 给变式题，引导识别"表面不同但原理相同" |
| 遗漏型 | 先补基础，再在 1/3/7 天后三次复习 |

## 跨会话模式识别

### 长期错误追踪

单次评估只看到"点"，跨会话才能看到"线"。当能够访问多次评估数据时，需执行以下分析：

#### 1. 顽固薄弱点识别
同一知识点在 ≥2 次评估中均表现为薄弱 → 标记为"顽固薄弱点"：
```markdown
⚠ 顽固薄弱点：[知识点] — 在 [日期1] 和 [日期2] 的评估中均属于薄弱环节，建议：
- 回到基础，用费曼技巧重新梳理概念
- 在接下来 1/3/7 天安排三轮针对性复习
- 更换讲解方式（如从类比型切换到图示型）
```

#### 2. 改善趋势判断
对比同一知识点在不同时间的评估结果：

| 趋势 | 表现 | 行动 |
|------|------|------|
| ↑ 改善 | 从薄弱→了解，或了解→掌握 | 标注"✅已改善"，当前方法有效 |
| → 持平 | 两次评估结果相同 | 调整教学方法或增练 |
| ↓ 退步 | 从掌握→了解，或了解→薄弱 | 标注"⚠退步"，检查是否存在基础断裂 |
| ✅ 已掌握 | 连续 2 次评估表现良好 | 移出重点复习区，进入间隔复习 |

#### 3. 系统性误解发现
当多个知识点在同一概念维度上出错 → 发现系统性误解：
```markdown
🔍 系统性误解：[概念维度] — 以下知识点均表现出同一类错误：[A], [B], [C]
→ 根本原因可能是[前置概念]未掌握
→ 建议：先补[前置概念]，再分别攻克各知识点
```

#### 4. 错误类型演化
追踪用户的最常见错误类型是否在变化：
- 初学期 → 应多为概念型/遗漏型
- 练习期 → 应转为程序型/迁移型
- 如果长期停留在概念型 → 说明基础加固不够

```markdown

| 知识点 | 上次结果 | 本次结果 | 趋势 |

```

## 关键原则

- 分离错误与自我价值——"错了说明你在尝试，很好"
- 找出根源，而非只给正确答案
- 一次不要修正太多——优先修正影响面最大的 1-2 个错误
- 跨会话追踪——单次错误可能是偶然，多次相同错误才是真正的薄弱点

## 反模式

- 直接给正确答案而不解释错误根因
- 将不同类型的错误用一个笼统的"不细心"来概括
- 仅指出错误，不提供纠正策略
