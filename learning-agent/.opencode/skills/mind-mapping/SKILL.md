# 思维导图方法

**来源**：Tony Buzan (1970s) 提出，后经认知科学研究验证其组织信息的有效性。

## 核心理念

将线性的文字转化为放射状的结构图。人的思维不是线性的，知识应该按关联而非顺序来组织。

## 文本版思维导图表示法

使用缩进 + 树形字符表示层级关系：

```
中心主题
├── 一级分支1
│   ├── 二级要点A
│   │   ├── 细节i
│   │   └── 细节ii
│   └── 二级要点B
├── 一级分支2
│   ├── 二级要点C
│   └── 二级要点D
└── 一级分支3
    └── 二级要点E
```

## 标注规则

| 符号 | 含义 |
|------|------|
| `→` | 指向/推导关系 |
| `~~` | 相关但非层级关系 |
| `★` | 核心考点/高频 |
| `⚠` | 易错/易混 |

## 层级规范

- 中心主题：一级（1 个）
- 一级分支：主要概念/大章节（3-7 个为佳）
- 二级分支：子概念/关键点
- 三级分支：细节/例子/公式
- 最深 4 级——超过 4 级说明需要重新组织

## 示例

```
导数
├── 定义
│   ├── 函数在某点的瞬时变化率
│   └── f'(x) = lim(h→0) [f(x+h)-f(x)]/h
├── 几何意义 ★
│   ├── → 切线斜率
│   └── ⚠ 导数存在 ↔ 可导 ↔ 该点有切线且非垂直
├── 求导法则
│   ├── 基本公式
│   │   ├── (xⁿ)' = nxⁿ⁻¹
│   │   ├── (sinx)' = cosx ~~ (cosx)' = -sinx
│   │   └── (eˣ)' = eˣ
│   └── 运算法则
│       ├── (u±v)' = u'±v'
│       ├── (uv)' = u'v + uv'
│       └── (u/v)' = (u'v-uv')/v²
└── 应用
    ├── → 求函数极值
    ├── → 判断单调性
    └── → 物理：速度和加速度的关系
```

## 在笔记中的应用

每当整理完一份笔记，在末尾添加思维导图大纲，方便用户快速回顾全文结构。

## 反模式

- 分支过多（一级分支超过 7 个，信息过载）
- 层级太浅（只有一二级，没有细节）
- 冗余重复（同一概念出现在多个位置）
