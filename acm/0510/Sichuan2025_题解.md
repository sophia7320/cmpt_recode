## A. Minimum Product

### 题目大意

给定一张有向图，有 $N$ 个点 $M$ 条边。每条边有两个权值属性 $(a_i, b_i)$。定义一条从点 $1$ 到点 $N$ 的路径 $P$ 的权值为 $(\sum_{i\in P} a_i) \times (\sum_{i\in P} b_i)$。求最小的路径权值，并输出对应的 $\sum a_i$ 和 $\sum b_i$。若有多个路径得到相同的最小权值，输出 $\sum a_i$ 最小的那组。

**数据范围**: $1 \leq N \leq 300$, $1 \leq M \leq 10^3$, $1 \leq a_i, b_i \leq 200$, 多组测试 $\sum N \leq 10^3$, $\sum M \leq 2 \times 10^3$。

### 解题思路

注意到 $a_i, b_i$ 都很小（$\leq 200$），且 $N \leq 300$，因此任意路径上 $\sum a$ 的最大值为 $300 \times 200 = 60000$。这提示我们可以用 DP 以 $\sum a$ 为状态。

**核心思想**: 设 $dp[v][x]$ 表示从点 $1$ 到点 $v$，边权和 $\sum a = x$ 时的最小 $\sum b$ 值。

由于是 DAG？不保证无环，但我们可以用类似 **Dijkstra** 的方式做 DP。具体来说，我们枚举每个点 $u$ 的出边 $(u,v,a,b)$，用 $dp[u][x] + b$ 更新 $dp[v][x+a]$。

由于边权非负，我们可以对每个 $\sum a$ 的值从小到大处理，类似分层图的最短路。

也可以直接用 **0-1 BFS** 或 **Dijkstra**，将状态 $(u, sum\_a, sum\_b)$ 视为图中的点，转移代价为边的 $a,b$。但这样状态太多。更好的做法是只记录 $sum\_b$ 对于每个 $sum\_a$ 的最小值，即 $dp[u][sum\_a] = \min(sum\_b)$。

转移：$dp[v][sum\_a + a] = \min(dp[v][sum\_a + a], dp[u][sum\_a] + b)$。

这可以看作是一个类似最短路的过程，由于 $a_i, b_i$ 都非负，我们可以用 BFS 或按 $sum\_a$ 递增的顺序 DP。

### 算法流程

1. 初始化 $dp[1][0] = 0$，其余 $dp[u][x] = \infty$
2. 对每条边 $(u,v,a,b)$ 进行松弛操作
3. 由于图可能不是 DAG，需要多次迭代直到收敛（最多 $N-1$ 次），或使用 SPFA 风格
4. 最终在所有 $x$ 中找使 $x \times dp[N][x]$ 最小的 $x$，若有多个取 $x$ 最小的

### 复杂度

- 状态数: $O(N \cdot M_a)$，其中 $M_a = \max \sum a \approx 60000$
- 转移: $O(N \cdot M_a \cdot \deg)$
- 实际可以用 Bellman-Ford 跑 $N$ 轮，总复杂度 $O(N \cdot M \cdot M_a)$，可能偏大但实际数据较小

更优的做法：注意到 $N \leq 300$，可以用 Floyd 风格 DP。定义 $f[x]$ 为到达 $N$ 且 $\sum a = x$ 的最小 $\sum b$，这样就可以倒着做 DP，但对所有 $v$ 都要考虑。

**优化**: 由于 $\sum a \leq 60000$ 不大，可以使用 **DP + 拓扑序**（若图有环则需要多次迭代）。实际上 $a,b \geq 1$，所以每条边都增大 $sum\_a$，这样按 $sum\_a$ 递增的顺序做 DP 即可避免环的问题，因为 $sum\_a$ 是严格单调递增的。

更准确的做法：直接做 DP $N$ 轮或直到收敛，因为 $a_i \geq 1$ 保证了 $sum\_a$ 递增。

### 实现要点

```cpp
// dp[v][x] = 到达 v 且 sum_a = x 的最小 sum_b
vector<vector<int>> dp(N+1, vector<int>(MAX_SUM, INF));
dp[1][0] = 0;

for (int sum_a = 0; sum_a < MAX_SUM; sum_a++) {
    for (int u = 1; u <= N; u++) {
        if (dp[u][sum_a] == INF) continue;
        for (auto [v, a, b] : g[u]) {
            if (sum_a + a < MAX_SUM)
                dp[v][sum_a + a] = min(dp[v][sum_a + a], dp[u][sum_a] + b);
        }
    }
}
// 最后找最小 product
long long best = INF;
int best_a = 0, best_b = 0;
for (int x = 0; x < MAX_SUM; x++) {
    if (dp[N][x] == INF) continue;
    long long prod = 1LL * x * dp[N][x];
    if (prod < best || (prod == best && x < best_a))
        best = prod, best_a = x, best_b = dp[N][x];
}
```

## B. Ternary

### 题目大意

**交互题**。有 $n$ 个三进制数字位 $i=0..n-1$，每个位对应一个双射 $f_i: \{0,1,2\} \to \{0,1,2\}$（即 6 种排列之一）。你可以询问最多 **2 次** 加法结果（在加密状态下）。每次询问给出两个 $n$ 位三进制数 $a,b$，返回解密后相加再加密的结果 $c$（共 $n+1$ 位，最高位是进位且不加密）。目标：恢复所有 $n$ 个 $f_i$。

### 解题思路

**关键观察**：三进制加法中，每个位的结果只依赖于该位的两个输入以及低位的进位。由于我们只有 2 次查询，需要精心设计查询内容。

对每个位置 $i$，加密 $f_i$ 将 $\{0,1,2\}$ 映射到 $\{0,1,2\}$。我们不知道哪个是 $0$，哪个是 $1$，哪个是 $2$。

**策略**:

1. **第一次查询**: 询问 $a = 00\ldots 0$ (全 0) 和 $b = 00\ldots 0$ (全 0)。这样可以得到每个位上 $0+0$ 的结果，从而知道 $f_i(0)$ 对应哪个值。更进一步，我们可以用非零数字来获取更多信息。

实际上，更有效的做法是：

- **第一次查询**: 对每个位放一个特定值，例如交替放 0 和 1，观察结果
- **第二次查询**: 改变输入来推导每个 $f_i$

但关键是：在三进制加法中，如果我们取 $a = 111\ldots 1$ 和 $b = 111\ldots 1$，则解密后 $f^{-1}(a) + f^{-1}(b)$ 每个位都是某个值加自身。

更系统的方法：

- 设 $f_i(0) = p_i$, $f_i(1) = q_i$, $f_i(2) = r_i$（未知的排列）
- 查询 1: 输入 $a = 000\ldots 0$, $b = 000\ldots 0$ → 返回的是每个位 $f_i(0) + f_i(0)$ 加密后的结果
- 由此可以知道每个位上哪个加密后的数字对应 $f_i(0)$（因为 $p_i + p_i$ 的进位信息）

实际上，我们需要更多信息来完全确定每个 $f_i$。

**标准解法**:

1. 查询 1: 输入全 0 + 全 0。返回结果告诉我们每个位的 $f_i(0) + f_i(0)$，从中可推断 $f_i(0)$（因为 $0+0=0$, $1+1=2$, $2+2=11$（三进制），不同的 $f_i(0)$ 产生不同的结果）。
2. 查询 2: 输入全 1 + 全 1。返回结果告诉我们 $f_i(1) + f_i(1)$，从中可推断 $f_i(1)$。
3. 剩下的 $f_i(2)$ 就是唯一剩下的值。

但这里有一个微妙之处：我们给交互器的输入是加密后的值，所以如果输入全 0，解密后可能变成任意值（取决于 $f_i^{-1}(0)$）。

**重新分析**: 输入是加密后的字符串。设输入为 $a$（加密后），则解密后为 $f^{-1}(a)$。我们需要设计加密后的输入。

**正确做法**:

- 查询 1: 输入 $a = 000\ldots 0$, $b = 000\ldots 0$。解密后 $f_i^{-1}(0) + f_i^{-1}(0)$ 的结果被加密后返回。
- 查询 2: 输入 $a = 111\ldots 1$, $b = 111\ldots 1$。同样分析。

通过分析每个位上两次查询返回的进位和结果，可以唯一确定 $f_i$。

具体来说，对于每个位置 $i$：

- 设 $x = f_i^{-1}(0)$，$y = f_i^{-1}(1)$
- 查询 1 得到 $f_i(x+x)$ 以及进到高位的进位
- 查询 2 得到 $f_i(y+y)$ 以及进位

### 算法流程

1. 读入 $n$
2. 查询 `? ` + $n$ 个 `0` + ` ` + $n$ 个 `0`
3. 读入结果 $c_1$ (长度为 $n+1$)
4. 查询 `? ` + $n$ 个 `1` + ` ` + $n$ 个 `1`
5. 读入结果 $c_2$ (长度为 $n+1$)
6. 对每个位置 $i$，分析 $c_1[i], c_1[i+1]$ 和 $c_2[i], c_2[i+1]$ 确定 $f_i$
7. 输出 `! ` + 所有 $f_i$

### 复杂度

$O(n)$ 每组数据，$\sum n \leq 10^6$。

## C. Optimal Time

### 题目大意

定义 $S(x) = \{d \mid d \text{ 是 } x \text{ 的因子}\} \cup \{kx \mid 2 \leq k \leq \lfloor N/x \rfloor\}$。即 $S(x)$ 是 $x$ 的所有因子和所有不超过 $N$ 的倍数的并集。

初始状态为 $x$。每秒有两种选择：

1. 以等概率随机跳到 $S(x)$ 中的一个数
2. 什么也不做

每秒结束时 $x \leftarrow x-1$。求在最优决策下到达状态 $0$ 的期望时间。多次查询。

### 解题思路

这是一个**马尔可夫决策过程**（MDP）。

定义 $E[x]$ 为从状态 $x$ 出发到达 $0$ 的最优期望时间。

状态转移方程：
$$E[x] = 1 + \min\left(E[x-1], \frac{1}{|S(x)|}\sum_{y \in S(x)} E[y]\right)$$

但注意，如果选择随机跳转，跳转后的状态 $y$ 可能是大于 $x$ 的数，然后再减 1！

**关键**: 由于 $x$ 每步减 1 是确定的，而随机跳可能是"好"的（跳到值更小、路径更短的状态），我们需要计算期望。

实际上，这个问题的结构暗示 $E[x]$ 关于 $x$ 是单调的：更大的 $x$ 需要更多步才能到 0。所以跳到更小的数是好的，跳到更大的数是不好的。

但 $S(x)$ 包含因子（都 $\leq x$）和倍数（都 $\geq x$）。跳到因子会减少状态，跳到倍数会增加状态。

**DP 方法**: 从 $x=1$ 到 $x=N$ 依次计算 $E[x]$。

$$E[x] = 1 + \min\left(E[x-1], \frac{x + \sum_{k=2}^{\lfloor N/x \rfloor} E[kx] + \sum_{d|x} (E[d] - E[d])}{|S(x)|} \right)$$

注意 $E[x]$ 依赖于 $E[x-1]$（更小）和 $E[kx]$（更大，$kx > x$）。这意味着需要迭代求解或从大到小计算。

实际上，由于这是期望最小化问题，且未来状态（$kx$）的期望可能比当前大，最优策略是：如果随机跳转的期望收益不如直接减 1，就不跳。

**简化分析**: 注意 $|S(x)|$ 可能很大。对每个 $x$，需要计算 $\sum_{y \in S(x)} \min(E[y], \ldots)$。

由于 $N \leq 10^5$，我们可以预处理每个数的因子和倍数。对每个 $x$，枚举 $S(x)$ 的每个元素并计算期望的调和平均。

实际上，这题的期望可以这样理解：在每个状态 $x$，如果选择随机跳，下一次跳转后的期望时间（包括跳到任意状态后的最优期望）为：

$$E_{jump}[x] = 1 + \frac{1}{|S(x)|} \sum_{y \in S(x)} E[y]$$

然后 $E[x] = \min(E[x-1] + 1, E_{jump}[x])$。

由于计算 $E[x]$ 需要知道 $E[y]$，且 $y$ 可能是 $x$ 的倍数（大于 $x$），我们需要按 $x$ 从大到小计算，这样 $kx$ 的 $E$ 值已经算出。

### 算法流程

1. 预处理每个 $x$ 的因子和倍数集合 $S(x)$
2. 从 $N$ 到 $1$ 计算 $E[x]$：
   - 如果 $x$ 选择随机跳，已经是全局最优策略就计算跳转后的期望
   - $E[x] = \min(1 + E[x-1], \text{跳转后的期望})$
3. 对每个查询直接输出 $E[x]$

### 复杂度

- 预处理: $O(N \log N)$ （枚举倍数）
- DP: $O(N \log N)$
- 查询: $O(1)$

## D. Tripartite Graph

### 题目大意

定义**三部图**：无向图能被 3 种颜色染色使得每条边的两端颜色不同。

给定排列 $q$，统计有多少排列 $p >_{lex} q$ 使得 $p$ 生成的图是三部图。$p$ 生成的图：对所有 $i<j$，若 $p_i > p_j$ 则连边 $(i,j)$。

求答案模 $998244353$。

### 解题思路

排列 $p$ 生成的图实际上是**排列的逆序图 (inversion graph / permutation graph)**。

一个排列的逆序图是三部图当且仅当排列可以被划分为 **最多 3 个递减子序列**（根据 Dilworth 定理，最长递增子序列长度 $\leq 3$）。

等价地，排列 $p$ **避免** pattern `1234`（即不存在 $i<j<k<l$ 且 $p_i < p_j < p_k < p_l$）。

这是因为排列的逆序图的色数 = 将排列划分为递减子序列的最小数 = 最长递增子序列的长度。

所以问题转化为：计数排列 $p >_{lex} q$ 且 $LIS(p) \leq 3$。

**计数方法**:

- 用 DP 逐位确定排列，同时维护已使用数字的状态
- 但由于 $n \leq 300$，需要更好的方法

**进一步转化**: 统计 $LIS \leq 3$ 的排列数 = 前 $n$ 个自然数中 123-avoiding 排列的计数，这是经典的 Catalan 数相关。

实际上，$LIS \leq 2$ 的排列（321-avoiding）是 $C_n$。$LIS \leq 3$ 的排列是 Baxter 排列等。可以用 DP 或 RSK（Robinson-Schensted-Knuth）对应计算。

**DP 方法**（$n \leq 300$）:
维护 DP 状态：已放置 $i$ 个元素，剩余可用的数字集合。但状态数太大。

更实用的方法：枚举第一个比 $q$ 大的位置，然后剩余部分自由排列，计数 $LIS \leq 3$ 的排列。

对每个前缀相同的 $i$，枚举 $p_i > q_i$ 的值，检查是否合法。

$LIS \leq 3$ 的排列数可以用 **Hook-length formula** 的推广。具体来说，$LIS \leq k$ 的排列数等于满足特定形状的 Standard Young Tableau 的数量。

对于 $k=3$，形状是行数 $\leq 3$ 的 Young diagram。排列数由 hook-length formula 给出：
$$\frac{n!}{\prod h(i,j)}$$

但需要限制行数 $\leq 3$。

**实用解法**: $n \leq 300$，可以 DP：
设 $dp[len][a][b][c]$ 表示当前长度为 $len$，最长递增子序列信息... 但这样太复杂。

**简化**: 使用 RSK 算法，$LIS \leq 3$ 的排列数等于行 $\leq 3$ 的 Young Tableau 的标准填充数。可以用 DP 枚举 Young diagram 形状计算。

但还有字典序约束 $p > q$，需要更细致的处理。

由于 $n \leq 300$ 较小，$O(n^3)$ 可行。可以用数位 DP 风格：从高位到低位确定 $p_i$，同时用数据结构维护 $LIS \leq 3$ 的约束。

### 算法概要

设 $dp[i][a][b]$ 表示已经处理了 $i$ 个位置（当前考虑的排列前缀），当前 $LIS$ 信息可以用两个值 $a,b$ 表示（如 patience sorting 中的堆顶）。

逐位 DP 计数，同时维护 $\leq 3$ 的限制。

### 复杂度

$O(n^3)$。

## E. Competition Graph

### 题目大意

竞赛图是一个有向完全图。给定 $n, k$，计数有多少个带标号的 $n$ 点竞赛图包含至少一个 $k$-环。答案模 $998244353$。

$2 \leq n, k \leq 10^5$。

### 解题思路

**关键性质**: 每个竞赛图都有一个**哈密顿路径**。竞赛图是强连通的当且仅当它有一个哈密顿环。实际上，强连通竞赛图包含所有长度从 3 到 $n$ 的环。

更精确的性质：

- 如果一个竞赛图包含一个 $k$-环，那么它包含所有长度 $3,4,\ldots,n$ 的环（如果它强连通）？不完全对。
- 已知：竞赛图的强连通分量形成一个全序（竞赛图的 condensation 是一个传递竞赛图）。
- 在一个强连通竞赛图中，每个顶点都属于至少一个 3-环（当 $n \geq 3$）。

**关键引理**: 竞赛图 $T$ 包含一个 $k$-环当且仅当它不是**传递的**。所有非传递竞赛图都包含至少一个 3-环。

实际上，任何非传递竞赛图必然包含一个 3-环。并且，如果竞赛图包含一个环，那么它包含 3-环。

但 $k$-环对 $k > 3$ 呢？不是说每个非传递竞赛图都有哈密顿环。实际上：

- 存在非传递竞赛图但没有 $k$-环的当 $k$ 较大时。
- 竞赛图中，存在 $k$-环 $\iff$ 强连通分量的大小分布满足一定条件。

**重点分析**: 

- 总竞赛图数: $2^{\binom{n}{2}}$
- 不含 $k$-环的竞赛图：这包括：
  1. 传递竞赛图（无环）: $n!$ 个
  2. 只有短环（长度 $< k$）的竞赛图

对于 $k > n$，答案显然是 0。

**当 $k \leq n$ 时**，不含 $k$-环的竞赛图：

- 传递竞赛图（无任何环）: $n!$ 个
- 且对于 $k=3$，非传递但无 3-环是不可能的。所以 $k=3$ 时答案是 $2^{\binom{n}{2}} - n!$。

对于 $k=4$：不含 4-环的竞赛图。这样的竞赛图有什么特征？任何长度 > 3 的环的存在性...

实际上，Landau 定理刻画了竞赛图的得分序列。Moon 定理：竞赛图是强连通的当且仅当没有非平凡的冠 (king)。

对于本题，$n,k \leq 10^5$，可能存在 $O(n)$ 或 $O(\log n)$ 的公式。

**重新思考**: 也许这题的目的是计算 2 的幂次减去传递竞赛图数。

不含任何环的竞赛图 = 传递竞赛图 = $n!$ 种（对应 $n$ 个顶点的全序）。

对于 $k \leq 3$：不含 3-环的竞赛图就是传递竞赛图。所以答案 = $2^{\binom{n}{2}} - n!$。

对于 $k > 3$：一个竞赛图可能包含 3-环但不包含 $k$-环。但这需要 $n$ 很小的情况，或者竞赛图结构受限制。

实际上，**竞赛图包含每个可能的环长**（从 3 到 $n$）当它是强连通时。所以不含 $k$-环的竞赛图必须满足其强连通分量的大小都 $< k$。

竞赛图的结构：将竞赛图分解为强连通分量，这些分量全序排列。设分量大小依次为 $c_1, c_2, \ldots, c_m$，则分量 $i$ 到分量 $j$ ($i<j$) 之间所有边方向一致。

不含 $k$-环等价于每个强连通分量的大小 $< k$。

### 算法

需要计数：$\sum_{c_1+\cdots+c_m=n, c_i < k} \prod \text{F}(c_i) \cdot 2^{...}$

其中 $F(c)$ 是大小为 $c$ 的强连通竞赛图的数量。

已知：$F(1)=1$, $F(2)=0$（两个顶点的竞赛图总是传递的），$F(c) = 2^{\binom{c}{2}} - \sum_{i=1}^{c-1} \binom{c}{i} F(i) 2^{\binom{c-i}{2}}$

这需要卷积/DP。

由于 $n \leq 10^5$ 且 $k$ 可能很大，可能需要分情况讨论或使用多项式技巧。

**简单情况**: $k=3$ 时，答案 = $2^{\binom{n}{2}} - n! \bmod 998244353$。

对于大 $k$，如果 $k > n$，答案为 0。如果 $k$ 较大但 $\leq n$，可以用 DP + 卷积。

## F. Inversion Pairs

### 题目大意

给定长度为 $n$ 的字符串 $s$，字符为 0、1 或 ?。你需要把 ? 替换为 0 或 1，最大化序列的逆序对数量。逆序对定义为 $i<j$ 且 $a_i > a_j$（即前面的 1 与后面的 0）。

$\sum n \leq 2 \times 10^6$。

### 解题思路

这实际上是一个经典的贪心问题。

对每个 ?，我们需要决定填 0 还是 1。决定的标准是考察它对逆序对总数的贡献。

设位置 $i$ 填 0 或 1：

- 填 0：与前面所有 1（包括已确定的 1 和前面 ? 填的 1）形成逆序对，贡献 = 前面的 1 的数量
- 填 1：与后面所有 0（包括已确定的 0 和后面 ? 填的 0）形成逆序对，贡献 = 后面的 0 的数量

直觉上是：把 ? 填成能产生更多逆序对的那个。

**更精确的分析**：
对序列从左到右扫描，每个 ? 位置，如果前面 1 的数量 < 后面 0 的数量，则填 1 更好（反之填 0）。

但是，? 之间的选择是相互影响的！一个 ? 填 1 会影响后面 ? 的"前面的 1 的数量"。

**最优策略**（经典结论）：
最优方案是：把所有 ? 放在前面的一部分填 1，剩下的填 0。分割点由贪心确定：扫描时，如果当前填 1 的收益（后面的 0 数量）大于填 0 的收益（前面的 1 数量），就填 1。

等价地：从左到右，维护一个平衡。对任意前缀，如果我们把一些 ? 变成 1，它们会增加后面所有 0（包括后面的 ? 变成的 0）的逆序对。

实际上，可以用前缀和和后缀和直接计算。

**有效算法**:

1. 预处理 $pre1[i]$ = 位置 $i$ 之前确定的 1 的数量
2. 预处理 $suf0[i]$ = 位置 $i$ 之后确定的 0 的数量
3. 把 ? 位置按出现顺序收集
4. 对于每个 ? 位置，可以计算如果填 0 或填 1 的临时贡献
5. 最优策略：前缀全填 1，后缀全填 0，最优分割点需要枚举

或者更简单的方法：对于所有 ?，先假设全填 1，计算逆序对数。然后考虑把某些 1 改成 0 的收益变化。

**O(n) 方法**:
从左到右扫描一次，维护"假设前面 ? 全填 1"时的前面 1 的数量（包括确定的 1 和假设的 ? → 1）。

然后从右到左扫描一次，对每个 ? 计算改为 0 的收益。

最优方案是存在一个分割点，前半部分的 ? 填 1，后半部分的 ? 填 0。用双指针或枚举分割点。

### 复杂度

$O(n)$ 每组数据。

## G. Diophantine Equation

### 题目大意

给定序列 $a_1,a_2,\ldots,a_n$ 和整数 $t$。求解 $x_1,\ldots,x_n$ 使得对 $j=0,1,\ldots,n-1$ 满足：
$$\sum_{i=1}^n a_i^j \cdot x_i \equiv t^j \pmod{998244353}$$
保证唯一解。

### 解题思路

这是线性方程组 $V \cdot x = T$，其中 $V_{j,i} = a_i^j$（$j$ 从 0 开始），$T_j = t^j$。

$V$ 是一个 **Vandermonde 矩阵**：
$$V = \begin{pmatrix}
1 & 1 & \cdots & 1 \\
a_1 & a_2 & \cdots & a_n \\
a_1^2 & a_2^2 & \cdots & a_n^2 \\
\vdots & \vdots & \ddots & \vdots \\
a_1^{n-1} & a_2^{n-1} & \cdots & a_n^{n-1}
\end{pmatrix}$$

而 $T = (1, t, t^2, \ldots, t^{n-1})^T$。

解这个方程组等价于求一个多项式 $P(z)$ 使得 $P(a_i) = ...$ 实际上：

观察 $x_i$ 的意义。考虑多项式插值：我们需要找到次数 $< n$ 的多项式 $P(z)$ 使得 $P(a_i) = \text{?}$

实际上，方程 $\sum x_i a_i^j = t^j$ 对所有 $j=0..n-1$ 成立。这意味着对于任意次数 $< n$ 的多项式 $Q(z) = \sum q_j z^j$，有：
$$\sum_i x_i Q(a_i) = Q(t)$$

特别地，取 $Q(z) = 1$ 得 $\sum x_i = 1$。

取拉格朗日基多项式 $L_i(z)$ = 在 $a_i$ 处为 1、其他 $a_j$ 处为 0 的 $n-1$ 次多项式。则：
$$x_i = L_i(t)$$

因为 $\sum_j x_j L_i(a_j) = x_i = L_i(t)$。

**所以**：
$$x_i = \prod_{j \neq i} \frac{t - a_j}{a_i - a_j} \pmod{M}$$

这就是拉格朗日插值公式！

### 算法

对每个 $i$，计算 $x_i = \frac{\prod_{j \neq i} (t - a_j)}{\prod_{j \neq i} (a_i - a_j)}$。

预处理：

- $P(z) = \prod_{j=1}^n (z - a_j)$
- 对每个 $i$，$x_i = \frac{P(t)}{(t-a_i)} / \prod_{j \neq i} (a_i - a_j)$

更高效的方法：

1. 计算总乘积 $P = \prod_{j=1}^n (t - a_j) \bmod M$
2. 对每个 $i$，$\text{den}_i = \prod_{j \neq i} (a_i - a_j) \bmod M$
3. $x_i = P \cdot (t - a_i)^{-1} \cdot \text{den}_i^{-1} \bmod M$

需要 $O(n)$ 或 $O(n \log n)$ 计算所有 $\text{den}_i$。可以用前缀/后缀积：

- $pref[i] = \prod_{j=1}^{i-1} (a_i - a_j)$
- $suf[i] = \prod_{j=i+1}^n (a_i - a_j)$
- $\text{den}_i = pref[i] \cdot suf[i]$

### 复杂度

$O(n)$。

## H. Hututu

### 题目大意

棋子初始在 $(x,y)$。每步可以移动到 $(x\pm1,y\pm1)$, $(x\pm2,y\pm2)$, $(x\pm1,y\pm2)$, $(x\pm2,y\pm1)$ 之一。求从 $(x,y)$ 到 $(X,Y)$ 的最少步数。

多组查询 $T \leq 10^6$，坐标范围 $[-10^9, 10^9]$。

### 解题思路

分析移动方式：

- $(1,1)$ 和 $(2,2)$ 型：沿对角线移动
- $(1,2)$ 和 $(2,1)$ 型：马的移动（国际象棋骑士）

所以这是一种**增强版的马**，额外增加了沿对角线的移动。

设 $dx = |X-x|$, $dy = |Y-y|$。由于对称，可以设 $dx \geq dy \geq 0$。

**贪心策略**:
马走 $(1,2)$ 可以减少 $dx$ 和 $dy$ 的差值。同时在差值为 0 时可以沿对角线走 $(1,1)$ 或 $(2,2)$。

1. 先用 $(1,2)$ 或 $(2,1)$ 缩小差距：
   - 每步减少 $dx$ 1 和 $dy$ 2（或反之）
2. 当 $dx$ 和 $dy$ 接近时，用 $(1,1)$ 或 $(2,2)$。

**更精确**:
设 $a = \max(dx, dy)$, $b = \min(dx, dy)$。

用 $(1,2)$ 和 $(2,1)$ 可以将 $(a,b)$ 同时减少。最优方式：

- 每 2 步可以分别减 2 和 2（先走 $(1,2)$ 再走 $(2,1)$）
- 先用 $(a+b)//3$ 步走 $(1,2)/(2,1)$ 型...

**公式化**:
对于马（无对角线移动），最少步数是 $\max(\lceil a/2 \rceil, \lceil (a+b)/3 \rceil, \text{加上奇偶性调整})$。

加上对角线移动后，公式变为：
$$ans = \max\left(\left\lceil \frac{a}{2} \right\rceil, \left\lceil \frac{a+b}{3} \right\rceil, \text{加上}(a+b)\text{奇偶性}\right)$$

更仔细的分析：我们可以在第一步尽量用 $(1,2)/(2,1)$ 减小差值，然后当 $a=b$ 时用 $(1,1)/(2,2)$。

**最终公式**（基于分析）:
设 $a = \max(|dx|, |dy|)$, $b = \min(|dx|, |dy|)$。

如果 $a \geq 2b$：答案 $= \lceil a/2 \rceil$

如果 $a < 2b$：答案 $= \lceil (a+b)/3 \rceil + ((a+b) \bmod 3 \text{ 的调整})$

实际上更简洁的公式：$\text{ans} = \max(\lceil (a+b)/3 \rceil, \lceil a/2 \rceil)$，并考虑当 $a$ 和 $b$ 同奇偶时的特殊情况。

具体公式可能需要微调，核心是解决 $O(1)$ 每组查询。

### 复杂度

$O(1)$ 每组查询，总 $O(T)$。

## I. Essentially Different Suffixes

### 题目大意

给定 $N$ 个字符串 $S_1, S_2, \ldots, S_N$。求所有字符串的所有后缀中，有多少个**本质不同**的后缀。

$\sum |S_i| \leq 3 \times 10^5$。

### 解题思路

这是经典的"多字符串的不同子串计数"问题，但只统计后缀。

**后缀**意味着每个后缀都是从某个位置开始到字符串末尾。对于单个字符串，不同后缀数量 = 字符串长度（因为每个后缀起始位置不同，但后缀内容可能重复）。

对于多个字符串，不同后缀的总数是所有后缀去重后的数量。

**方法 1: 后缀自动机（SAM）**

广义后缀自动机（Generalized SAM）可以处理多个字符串。将所有字符串插入 SAM 中，每个节点代表一个等价类。

构建 GSAM 后，每个节点 $v$ 的 $\text{len}(v) - \text{len}(\text{link}(v))$ 之和就是不同子串数。但这里我们只关心后缀。

对于后缀，需要特殊处理。每个字符串的每个后缀对应 GSAM 中的一条路径。不同后缀数量 = GSAM 中所有状态表示的、且作为某个字符串后缀出现的子串数量。

实际上，构建 GSAM 时，将每个字符串逐个插入。对于每个字符串的每个前缀，在 SAM 中有一个对应状态。后缀就是从这些状态出发到根路径上的所有子串。

**方法 2: 后缀数组 (SA)**

将所有字符串用不同的分隔符连接，构建后缀数组 + LCP。不同后缀数量 = 所有后缀数 - 重复后缀数（由 LCP 给出）。

**更简单的理解**:
将所有字符串用特殊分隔符连接为一个大字符串（分隔符必须各不相同且不在字母表中）。在这个大字符串上统计不同后缀。

由于分隔符不会出现在原字符串中，大字符串的每个后缀唯一对应原来某个字符串的某个后缀。但问题是分隔符会导致一些后缀跨越了原字符串边界。

实际上，更直接的做法是：构建**广义后缀自动机**。插入第一个字符串，记录其每个前缀对应的 SAM 状态。然后插入第二个，同样记录。最后统计所有被至少一个字符串标记为"后缀"的不同子串。

### 算法（GSAM）

1. 构建 GSAM，每个状态维护一个掩码（或布尔标记）表示哪些字符串的后缀经过该状态。
   - 插入时，对于每个前缀对应的状态（last），将该状态标记为对应字符串的后缀终点
2. 按长度从大到小处理状态，将后缀标记向 link 传播
3. 统计：对所有状态，若被 $\geq 1$ 个字符串标记，贡献 = $\text{len}(v) - \text{len}(\text{link}(v))$

### 复杂度

$O(\sum |S_i| \cdot |\Sigma|)$，其中 $|\Sigma| = 26$。

## J. Sichuan Provincial Contest

### 题目大意

给定一棵 $n$ 个节点的树，每个节点上有一个大写字母 $S_i$。求有多少条包含恰好 5 个节点的简单路径 $u,v,x,y,z$ 使得 $S_u S_v S_x S_y S_z$ = `SCCPC`。

多次测试 $\sum n \leq 2 \times 10^6$。

### 解题思路

路径中 5 个节点顺序已确定，且路径中节点必须是连续的（相邻节点在树上有边相连）。

需要统计长度为 4（5 个节点）的简单路径，其字符序列恰好是 `SCCPC`。

**树形 DP**:
在树上对路径进行计数。关键观察：路径 `SCCPC` 长度为 5。

枚举路径的"中心"位置（第 3 个位置 `C`，即 $x$），然后向两边扩展。

对于节点 $x$（字符为 `C`）：

- 需要选择前两个节点：$u$（字符 `S`）和 $v$（字符 `C`），且路径为 $u \to v \to x$
- 需要选择后两个节点：$y$（字符 `P`）和 $z$（字符 `C`），且路径为 $x \to y \to z$

且整个路径 $u \to v \to x \to y \to z$ 必须是简单路径（不自交）。

这意味着 $v$ 和 $y$ 必须在 $x$ 的**不同**子树中（如果 $v$ 和 $y$ 在相同子树，则路径会自交）。而 $u$ 在 $x$ 或 $v$ 的方向上，$z$ 在 $x$ 或 $y$ 的方向上。

**分情况讨论**: 路径的形状可以是：

1. $u \to v \to x \to y \to z$ 是一条链（5 个不同的方向分支）
2. $u = x$（即开头 3 个节点有重叠）等

但题目要求 5 个节点不同（简单路径），所以 5 个节点必须是树中不同的节点。

**DP 状态设计**:
对于每个节点，统计：

- $cnt[0]$: 以该节点为终点，长度为 1 的路径，字符为 `S` 的数量（即邻接的 `S` 字符节点数）
- $cnt[1]$: 以该节点为终点，长度为 2 的路径，字符序列为 `SC` 的数量
- ...

实际上，由于字符串固定为 `SCCPC`，我们可以在树上做 DP：

- 对于每个节点 $x$（字符 `C` - 第 3 个位置的 C）：
  - 计算以 $x$ 为中心，前后缀匹配 `SC` 和 `PC` 的方案数
  - 向前缀方向：在 $x$ 的子树中统计从 `S` 开始的、中间经过 `C` 到 $x$ 的路径数
  - 向后缀方向：从 $x$ 出发经过 `C` 到 `P` 再到 `C` 的路径数

**具体解法**:
由于字符串很短（长度 5），可以暴力枚举所有可能的路径中心并累加。

以 $x$（第3个 `C`）为中心：

- 需要在 $x$ 的邻居中找到匹配的路径段
- $x$ 的度数为 $\deg(x)$
- 对每个方向（每个邻居），统计该方向上有多少路径匹配前缀 `SC` 和后缀 `PC`

关键：统计每个方向上的路径信息，然后组合不同方向。

**实现方案**:

1. 枚举 $x$（第3个位置），字符必须是 `C`
2. 对 $x$ 的每个邻居 $nbr$，在该邻居方向上匹配：
   - 匹配 `S-C`（$nbr$ 是 `C`，$nbr$ 的有字符 `S` 的邻居数）
   - 匹配 `C-P-C`（$nbr$ 是 `P`，$nbr$ 的有字符 `C` 的邻居方向上的 `C` 数更复杂）
   - 或 $nbr$ 是 `P`，其邻居有是 `C` 的，再下一层是 `S`

实际上，我们需要对每个节点做 DP：

```
dp0[x] = 1 if S[x] == 'S'
dp1[x] = sum_{y in adj[x]} dp0[y] if S[x] == 'C'
dp2[x] = sum_{y in adj[x]} dp1[y] if S[x] == 'C'  (第3个位置)
dp3[x] = sum_{y in adj[x]} dp2[y] if S[x] == 'P'
dp4[x] = sum_{y in adj[x]} dp3[y] if S[x] == 'C'  (第5个位置)
```

但问题是路径可能回头（如 $u=x$ 和 $z$ 在同一方向等），需要减去非简单路径。

一种更安全的方法是枚举 $x$（第3个位置 `C`），在其邻居中选择两个**不同的**方向，一个向前走 `S-C`，一个向后走 `P-C`，然后组合。

### 复杂度

$O(n)$ 每组数据，总 $O(\sum n)$。

## K. Point Divide and Conquer

### 题目大意

给定一棵无根树 $T$（$n$ 个顶点）和一个排列 $p_1,\ldots,p_n$。按以下操作构建有根树 $T'$：

1. 在 $T$ 中找出在排列 $p$ 中最早出现的顶点 $x$
2. 从 $T$ 中移除 $x$，将 $x$ 作为 $T'$ 的根
3. 剩余顶点形成若干连通分量 $T_1,\ldots,T_k$
4. 对每个 $T_i$ 递归执行操作得到 $T_i'$
5. 将 $T_i'$ 加入 $T'$，设 $T_i'$ 的根的父节点为 $x$

求最终 $T'$ 中每个顶点的父节点。

$\sum n \leq 10^6$。

### 解题思路

这个过程本质上是在反复"去除"排列中最早出现的节点。这个过程非常类似 **Cartesian 树** / **Treap** 的构建，但针对的是树上的"分治"。

换个角度思考：按照排列 $p$ 中**从后往前**的顺序，依次向森林中添加节点：

- 当添加节点 $v$ 时（按 $p$ 从后往前的顺序），$v$ 的父节点是在当前已添加的节点中，与 $v$ 在原树中相邻且最早被添加到森林中的那个节点。

**并查集方法**（正序）:
按 $p_1, p_2, \ldots, p_n$ 的顺序处理每个顶点。当处理到顶点 $v$ 时：

1. 标记 $v$ 为"已访问"
2. 在原树中，如果 $v$ 的邻居 $u$ 已经被访问过（之前处理过），则 $u$ 所在的连通分量中最早被处理的节点（即该分量的"根"）的父节点设为 $v$。
   - 用并查集维护连通分量，每个分量的根是该分量中在 $p$ 中出现最早的节点。

**实现细节**:

- 按 $p$ 的顺序遍历顶点
- 维护 `vis[v]` 表示 $v$ 是否已处理
- 维护 `parent[v]` 表示 $v$ 在 $T'$ 中的父节点
- 使用并查集维护每个连通分量的"根"（该分量中在 $p$ 中出现最早的节点）
- 当处理 $v$ 时，对于 $v$ 的每个邻居 $u$：
  - 如果 $u$ 已被处理，找到 $u$ 所在分量的根 $r$，设 $\text{parent}[r] = v$
  - 然后将 $u$ 的分量与 $v$ 合并，新根为 $v$

这个算法的工作过程模拟了"点分治"的递归分解：

- 先选的节点是"分治中心"
- 它把树分成若干连通块
- 每个连通块的"根"是其内部最早被选的节点
- 该连通块的根连接到上一层分治中心

### 算法

```cpp
// pos[v] = 节点v在排列p中的位置 (1-indexed)
// 按p顺序处理节点: for id in 1..n: v = p[id]

DSU dsu(n);
vector<int> parent(n+1, 0);
vector<bool> vis(n+1, false);

for (int i = 1; i <= n; i++) {
    int v = p[i];
    vis[v] = true;
    for (int u : adj[v]) {
        if (vis[u]) {
            int r = dsu.find(u);  // u所在分量中最早出现的节点
            parent[r] = v;
            dsu.union(u, v, v);   // 合并，新根为v
        }
    }
}
```

注意：并查集在 union 时需要保证新根是 $v$（因为 $v$ 是刚加入的，且在 $p$ 中出现比之前的任何节点都晚）。

### 复杂度

$O(n \cdot \alpha(n))$。

## L. abc

### 题目大意

给定一个长度为 $N$ 且仅包含字符 `a`, `b`, `c` 的字符串 $S$。

定义字符串 $S$ 的"价值" $\text{val}(S)$ 为：出现次数最多的字符的出现次数 减去 出现次数最少的字符的出现次数。

注意：最少的出现次数不能为 0——即只统计实际出现过的字符中最少的次数。

例如：`val(aaa) = 0`（只有 a），`val(aab) = 2 - 1 = 1`，`val(abccc) = 3 - 1 = 2`。

求所有子串的 $\text{val}$ 之和：$\sum_{i=1}^{N} \sum_{j=i}^{N} \text{val}(S[i..j])$。

**数据范围**: $1 \leq N \leq 2 \times 10^5$。

### 解题思路

子串中只有 3 种字符，分情况讨论：

1. **子串只含 1 种字符**：$\text{val} = 0$（最多的 = 最少的），无贡献。
2. **子串含恰好 2 种字符**：设出现次数为 $c_1, c_2$，则 $\text{val} = |c_1 - c_2|$。
3. **子串含 3 种字符**：设出现次数为 $c_a, c_b, c_c$，则 $\text{val} = \max(c_a,c_b,c_c) - \min(c_a,c_b,c_c)$。

**核心思路**：枚举子串中"最多的字符"是哪一个，计算相应的贡献。

设 $p_a[i]$ 表示前缀 $[1..i]$ 中字符 $a$ 的出现次数（同理 $p_b, p_c$）。

对于子串 $[l+1..r]$（$0 \leq l < r \leq N$），有：

- $c_a = p_a[r] - p_a[l]$
- $c_b = p_b[r] - p_b[l]$
- $c_c = p_c[r] - p_c[l]$

**方法：贡献法**

对于每个子串，$\text{val}(S[l..r]) = \max(c_a,c_b,c_c) - \min(c_a,c_b,c_c)$。

定义 $f(l,r) = \max(p_a[r]-p_a[l], p_b[r]-p_b[l], p_c[r]-p_c[l]) - \min(p_a[r]-p_a[l], p_b[r]-p_b[l], p_c[r]-p_c[l])$。

由于只有 3 种字符，我们可以枚举"最多种"和"最少的"分别是哪两种，然后计数满足条件的 $(l,r)$ 对。

**关键转化**：对于固定的 $(l,r)$，设三种字符的增量为 $d_a = p_a[r]-p_a[l]$, $d_b = p_b[r]-p_b[l]$, $d_c = p_c[r]-p_c[l]$。

则 $\max - \min = \max(d_a,d_b,d_c) - \min(d_a,d_b,d_c)$。

这等于 $\max(|d_a-d_b|, |d_a-d_c|, |d_b-d_c|)$。

所以 $\sum \text{val} = \sum_{l<r} \max(|d_a-d_b|, |d_a-d_c|, |d_b-d_c|)$。

进一步，$\max(x,y,z) = x + y + z - \min(x,y) - \min(x,z) - \min(y,z) + \min(x,y,z)$。

应用这个公式（当 $x,y,z$ 中至少有两个等于最小值的那个为 0）：

$$\text{val} = |d_a-d_b| + |d_a-d_c| + |d_b-d_c| - \min(|d_a-d_b|, |d_a-d_c|) - \min(|d_a-d_b|, |d_b-d_c|) - \min(|d_a-d_c|, |d_b-d_c|) + \min(|d_a-d_b|, |d_a-d_c|, |d_b-d_c|)$$

其中 $\min(|d_a-d_b|, |d_a-d_c|)$ 实际上返回的是中间的那个差值。

化简：$\text{val} = \text{最大的差值}$。

这是因为对于三个数 $A,B,C$，$\max-\min = \max(|A-B|, |A-C|, |B-C|)$，且按上述公式展开后恰等于最大差值。

**更实用的方法**：

由于只有 3 种字符，枚举子串中"缺少哪类字符"：

- **缺 c**（只有 a,b 出现）：$\text{val} = |d_a - d_b| = |(p_a[r]-p_b[r]) - (p_a[l]-p_b[l])|$
- **缺 b**：$\text{val} = |d_a - d_c|$
- **缺 a**：$\text{val} = |d_b - d_c|$

定义 $A[i] = p_a[i] - p_b[i]$, $B[i] = p_a[i] - p_c[i]$, $C[i] = p_b[i] - p_c[i]$。

对于只含 a,b 的子串：$\text{val} = |A[r] - A[l]|$

对于只含 a,c 的子串：$\text{val} = |B[r] - B[l]|$

对于只含 b,c 的子串：$\text{val} = |C[r] - C[l]|$

对于含三种字符的子串：$\text{val} = \max(|A[r]-A[l]|, |B[r]-B[l]|, |C[r]-C[l]|)$。

**整体算法**：

1. 计算前缀数组 $A, B, C$
2. 对每种"字符出现情况"分别计数：
   - 只有 a,b：找到所有 $l,r$ 使得 $c_c$ 在 $[l+1..r]$ 中为 0。对这样的区间，$\text{val} = |A[r]-A[l]|$。可以用类似"c 作为分隔符"的方式，在相邻 c 之间的段中计算贡献。
   - 类似处理其他两种情况。
   - 含三种字符：需要计算 $\max(|A[r]-A[l]|, |B[r]-B[l]|, |C[r]-C[l]|)$。

对于情况 4（三种字符都出现），需要注意避免重复计算（因为一个子串可能同时属于"缺 c"和"缺 a"等，但实际上"三种字符都出现"是互补的）。

更好方法：先统计所有子串的总 $\text{val}$，然后减去只有 1 或 2 种字符的子串的 $\text{val}$（需要修正）。

**实用解法**（基于前缀和的扫掠线）：

枚举右端点 $r$，用数据结构维护以 $r$ 为右端点的所有子串的贡献。

由于字符种类只有 3，状态空间很小。对于每个 $l$，子串 $[l+1..r]$ 的字符种类由 $l$ 和 $r$ 之间每种字符是否出现过决定。

我们可以维护每种字符最近出现的位置 $last_a$, $last_b$, $last_c$（相对于当前 $r$）。

对于当前的 $r$：

- 如果 $l \geq last_a$，则子串不含 a
- 如果 $l \geq last_b$，则子串不含 b
- 如果 $l \geq last_c$，则子串不含 c

设 $m_1 = \max(last_a, last_b, last_c)$, $m_2 = \text{median}$, $m_0 = \min(last_a, last_b, last_c)$。

则 $l$ 分 4 段：

1. $0 \leq l < m_0$：三种字符都有
2. $m_0 \leq l < m_2$：缺一种（最早出现的那一种）
3. $m_2 \leq l < m_1$：缺两种（只有最晚出现的那一种）
4. $m_1 \leq l < r$：只有一种字符

对每段分别计算 $\sum \text{val}$。

这可以用 **线段树或 BIT** 维护前缀和。

在第 k 段中，子串的差值公式可以化简。例如，对于只有 a 和 b 出现的区间，$\text{val} = |(p_a[r]-p_b[r]) - (p_a[l]-p_b[l])| = |A[r] - A[l]|$。

我们需要快速计算：对于给定的 $l$ 区间 $[L, R)$，$\sum_{l=L}^{R-1} f(A[l], B[l], C[l])$ 依赖于区间内字符出现情况。

对于 1 种字符的情况：$\text{val} = 0$。
对于 2 种字符的情况：直接用 $A$ 数组。
对于 3 种字符的情况：$\max(|A|,|B|,|C|)$。

**最终算法**（$O(N \log N)$）:

从左到右扫掠右端点 $r$，维护：

- 三种字符最近出现的位置
- 用 3 个 BIT/Fenwick 树维护前缀数组 $A,B,C$ 的排序后信息

对于每个 $r$，按照 $l$ 的 4 段区间分别计算贡献。

由于本题数据范围 $N \leq 2 \times 10^5$，这样可以通过。

**简化版解法**（基于贡献分解）:

$\text{val}$ 实际上等于三个差值中最大的那个。我们可以：

1. 分别计算三个差值的绝对值之和在所有子串上的总和
2. 再加上三个最大值的贡献

对于"总所有子串中 $\max(|A|,|B|,|C|)$ 的和"，由于只有三种字符，我们可以用容斥原理。

### 复杂度

$O(N \log N)$ 或 $O(N)$（取决于具体实现）。

### 参考

此题类似 AtCoder ABC 中的经典问题，核心技巧包括前缀和 + 线段树 / BIT + 分类讨论。关键是要将 $\text{val}$ 公式分解为可独立计算的贡献。
